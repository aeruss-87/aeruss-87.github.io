EA Weekly Pulse

A Claude skill that reads your executive's calendar, email, and Slack, then generates a weekly dashboard covering every meeting, travel detail, open thread, and prep gap.

What it generates
📅

This Week

Every meeting, chronological, with attendee count, location, and flagged conflicts.

✈️

Travel

Flights and hotels pulled from the calendar by flight number or confirmation code.

💬

Open Threads

Email and Slack threads your exec was asked something in and hasn't replied to.

📋

Needs Prep

Multi-attendee meetings with no prep block in the 48 hours before them.

Works with whatever tools you have

On first run, the skill walks you through a short setup: your exec's name and email, which calendar and email systems you use, whether Slack is connected, which channels to monitor, and how many days ahead to look. Your answers are saved locally so every future run is one command. The skill only uses the tools you confirm you have connected — if you skip Gmail or Slack, those sections are simply omitted from the dashboard.

Installation
1

Download the skill file

Open Terminal and run this command. It creates the skills folder and downloads the SKILL.md file into the right place.

mkdir -p ~/.claude/skills/ea-weekly-pulse && curl -o ~/.claude/skills/ea-weekly-pulse/SKILL.md https://aeruss-87.github.io/ea-pulse-setup/SKILL.md

Paste this as one line. It works on Mac and Linux.

2

Connect your MCP servers in Claude Code

In Claude Code, open Settings and connect the MCPs for the tools you use. At minimum, connect Google Calendar. Gmail and Slack are optional but unlock the Open Threads section.

If you are not sure how to add an MCP, type /update-config in Claude Code and ask it to help you add the Google Calendar MCP.

3

Run the skill for the first time

Open Claude Code and type:

/ea-weekly-pulse

Claude will ask you 8 setup questions (your exec's name, email, which tools you use, which Slack channels to monitor, how many days to look ahead). This only runs once. After that, just type /ea-weekly-pulse and the dashboard generates automatically.

4

Open your dashboard

When it finishes, a file named ea-pulse-[date].html appears on your Desktop. Open it in any browser. Four tabs: This Week, Travel, Open Threads, and Needs Prep.

Common questions

How do I reset or change my setup answers?

Delete the config file by running rm ~/.claude/ea-pulse-config.json in Terminal. The next time you run /ea-weekly-pulse, it will walk you through setup again.

I manage multiple executives. Can I run this for each one?

Not at the same time with one config file. For now, reset the config to switch execs. A multi-exec version is a potential future update.

Can I schedule it to run automatically every Monday?

Yes. After the skill runs, it will offer to set this up. Type /schedule in Claude Code and describe the task as: "Run /ea-weekly-pulse every Monday at 8am."

What if I don't have Gmail or Slack connected?

The skill skips those sections entirely and tells you what was and wasn't checked. You still get a full calendar view, travel items, and prep gaps from Google Calendar alone.

Does this send anything anywhere?

No. Everything stays local. The dashboard is a plain HTML file on your Desktop. Nothing is posted, emailed, or synced automatically.
